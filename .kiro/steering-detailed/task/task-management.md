# タスク管理共通ルール

## 🎯 基本方針

全プロジェクトで一貫したタスク管理を行い、進捗の可視化と効率的な開発を実現します。

## 📋 タスク管理フロー

### 1. アイデア段階
**ファイル**: `future-ideas.md`

- 新機能のアイデアはまず`future-ideas.md`に追加
- アイデアの検討・熟成期間を設ける
- 実装に値するかどうかを判断

**フォーマット**:
```markdown
## [IDEA-XXX] アイデア名

**概要**: アイデアの簡潔な説明

**背景**: なぜこのアイデアが必要か

**期待効果**: 実装した場合の効果

**実装難易度**: 低/中/高

**優先度**: 低/中/高

**ステータス**: 検討中/実装決定/見送り/実装済み
```

### 2. 実装決定
**ファイル**: `implementation-tasks.md`

- 実装を決定したアイデアを`TASK-XXX`として追加
- 優先度、見積もり、背景を記載
- タスク分解を行う

**フォーマット**:
```markdown
## [TASK-XXX] タスク名

**概要**: タスクの簡潔な説明

**背景**: 実装する理由・背景

**受入基準**: 完了の定義

**見積もり**: 工数（時間）

**優先度**: 高/中/低

**依存関係**: 他のタスクとの関係

**ステータス**: 未着手/進行中/完了

**進捗率**: 0% / 25% / 50% / 75% / 100%

**完了日**: YYYY-MM-DD (vX.X.X)
```

### 3. 進捗管理

#### 進捗率の更新
- サブタスク完了時に即座に更新
- 25%刻みで進捗を記録
- 完了時に完了日とバージョンを記録

#### ステータス管理
- **未着手**: まだ開始していない
- **進行中**: 作業中（進捗率1-99%）
- **完了**: 実装・テスト・ドキュメント全て完了

## 📊 タスク分類

### 優先度分類
- **高**: 緊急性・重要性が高い
- **中**: 通常の機能追加・改善
- **低**: あると良い機能・将来的な改善

### 工数見積もり
- **小**: 1-4時間
- **中**: 4-16時間
- **大**: 16時間以上

### タスクタイプ
- **feat**: 新機能
- **fix**: バグ修正
- **docs**: ドキュメント
- **refactor**: リファクタリング
- **test**: テスト追加・改善

## 🔄 タスクライフサイクル

### 1. タスク作成
```
future-ideas.md → implementation-tasks.md
```

### 2. 実装フェーズ
```
未着手 → 進行中 (進捗率更新) → 完了
```

### 3. アーカイブ
```
implementation-tasks.md → completed-tasks.md
```

## 📈 進捗追跡

### 日次進捗確認
- 進行中タスクの進捗率確認
- ブロッカーの特定と解決
- 次の作業項目の明確化

### 週次レビュー
- 完了タスクの振り返り
- 未完了タスクの見直し
- 優先度の再評価

### 月次アーカイブ
- 完了タスクのアーカイブ
- 長期未着手タスクの見直し
- 新規アイデアの評価

## 🎯 プロジェクト固有の調整

各プロジェクトは以下を調整可能：
- タスク番号の開始番号
- 進捗率の刻み（25%/10%等）
- 優先度の基準
- 見積もりの単位
